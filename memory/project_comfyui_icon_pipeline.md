# ComfyUI icon pipeline notes

- `project/tools/comfyui/icon_img2img.py` writes intermediates and final icon variants to `C:\Users\Shadow\Desktop\Working Files\Dropzone\comfyui` by default.
- Override the output location with `CSC_COMFYUI_OUTPUT_DIR` when needed.
- `icon_postprocess.py` should create rembg with `new_session("u2net", providers=["CPUExecutionProvider"])`; the Windows Python environment has `onnxruntime-gpu` installed but CUDA 12 DLLs are not on PATH, causing noisy CUDA provider errors if rembg probes default providers.
- `sam_outline.py` is optional. If absent or broken, the supported fallback is silhouette-only outlining without printing a scary stack/error message.
- ComfyUI can report a fully cached successful run with an empty `outputs` block when the workflow is identical. `icon_img2img.py` avoids this by making the `SaveImage.filename_prefix` unique per run and retrying once if history has no images.
- ComfyUI input uploads use a unique `RUN_ID` for the main source image to avoid stale/ambiguous uploads. The Canny guide is intentionally always written/uploaded as `canny_input.png` so the output folder does not accumulate one Canny file per run.
- The icon img2img path assumes transparent input means intentional silhouette: Canny is alpha-aware, generated RGB is masked back to the input alpha before saving the raw output, and postprocess skips rembg when raw input already has transparency. Final output filenames are derived from the input filename by replacing `Input` with `Output` (e.g. `CSC_BAKERS_Wind_Mill_Output.png`). Default output directory for both `icon_img2img.py` and `icon_postprocess.py` is `C:\Users\Shadow\Desktop\Working Files\Icons\Buildings\ComfyUI output`, overridable with `CSC_COMFYUI_OUTPUT_DIR`. `icon_postprocess.py` has top-of-file master toggles for each effect: background removal, rembg padding, Canny crease overlay, structural outlines, SAM outlines, silhouette outline, and canvas centering; each can also be overridden with `CSC_ICON_ENABLE_*` env vars. SAM outlines are implemented in `project/tools/comfyui/sam_outline.py` using `segment_anything` vit_b; checkpoint lives outside the repo at `C:\Users\Shadow\.cache\civ_supply_chains\sam\sam_vit_b_01ec64.pth`, overridable with `CSC_ICON_SAM_CHECKPOINT`. Outline thickness is controlled by `OUTLINE_THICKNESS` / env var `CSC_ICON_OUTLINE_THICKNESS`; outline hug/tightness is controlled by `OUTLINE_ALPHA_THRESHOLD` / env var `CSC_ICON_OUTLINE_ALPHA_THRESHOLD` (default 160) so the silhouette outline anchors to the solid subject instead of rembg's low-alpha fringe. Definition can come from Canny crease overlay (`CSC_ICON_CREASE_OPACITY`, `CSC_ICON_CREASE_WIDTH`, `CSC_ICON_CREASE_THRESHOLD`) and/or structural outlines when enabled. Color grading is enabled by default from the Market_Output.xcf reference effects: midtone color balance green +0.054 / blue +0.015, saturation scale 0.884, levels gamma 0.79, color temperature 6500K→6108.6K; knobs are `CSC_ICON_COLOR_BALANCE_MID_GREEN`, `CSC_ICON_COLOR_BALANCE_MID_BLUE`, `CSC_ICON_COLOR_SATURATION_SCALE`, `CSC_ICON_COLOR_LEVELS_GAMMA`, `CSC_ICON_COLOR_TEMP_ORIGINAL`, `CSC_ICON_COLOR_TEMP_INTENDED`, with master toggle `CSC_ICON_ENABLE_COLOR_CORRECTION`. Canny and alpha masks are pre-scaled with ComfyUI's exact center-crop math to 1024x1024 before generation/overlay to avoid alignment drift. For opaque inputs, `icon_postprocess.py` pads before rembg, so the Canny overlay must be pasted into the same padded canvas at `pad_amount` before centering; resizing the Canny across the padded image causes scale/offset drift.
